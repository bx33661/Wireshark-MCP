---
title: 证据标准
description: 编写可复现的数据包分析结论。
---

Wireshark MCP 报告应该区分事实和解释。事实来自工具输出。解释说明事实意味着什么，以及分析者有多大把握。

## 置信标签

| 标签 | 含义 |
| --- | --- |
| `confirmed` | 包证据直接支持结论。 |
| `likely` | 证据强，但抓包限制导致无法完全证明。 |
| `possible` | 行为可疑或相关，但仍存在常见良性解释。 |
| `unresolved` | 抓包不包含足够证据来判断。 |

## 必要证据

非平凡发现至少包含其中两项：

- 发现线索的工具调用
- 精确主机、端口、协议或字段名
- 流索引或帧号
- 显示过滤器或提取查询
- 解释该信号为什么重要

## 良好发现结构

```text
Finding: 观察到明文 FTP 凭据。
Confidence: confirmed
Evidence: wireshark_extract_credentials 在 stream 4 中报告 FTP USER/PASS。
Frames: 182, 185
Impact: 拥有抓包访问权限的人可能恢复凭据。
Next step: 轮换暴露账号，并检查后续会话是否复用该账号。
```

## 避免这些错误

- 没有包证据就把少见流量称为恶意
- 因为载荷加密就把流量称为良性
- 过度信任单一启发式规则
- 把专家信息当作结论
- 忽略抓包视角限制
- 不说明缺少什么证据
