当抓包属于调查材料，答案需要时间线、影响范围和证据链时，使用这个流程。

## 推荐流程

1. 先执行快速分析流程。
2. 用 `wireshark_get_file_info` 确认抓包时长和边界。
3. 用 `wireshark_list_ips`、`wireshark_stats_endpoints`、`wireshark_stats_conversations` 绘制参与方。
4. 运行安全工具识别 IOC 和可疑流量。
5. 跟踪与初始联系、凭据使用、横向移动、下载、上传或命令流量相关的流。
6. 用帧号和时间戳锚定关键事件。

## 解读提示

抓包开始和结束时间不等于事件开始和结束时间。当抓包窗口不足以证明初始访问或最终影响时，需要明确说明。

## 报告结构

包含：

- 事件时间线
- 受影响主机和服务
- IOC 列表
- 可能的攻击叙事
- 遏制或后续调查建议
